RESPONSIVE DESIGN
Web design that looks great and remains interactive on all devices, not matter the screen size.
MANSORY DESIGN
A fluid, tiled web design that efficiently displays information without overcluttering the website. From Pintrest to Google Plus, lots of platforms use this design to display feed in an appealing, but efficient way.
MATERIAL DESIGN
The web design that Google documents and uses in its web applications. Material design focuses on the user expirience, and aims to make website interfaces have a common, but elegant theme throughout internet. This design outlines how almost every website element should be implemented, from tables and icons to forms.
WELL DOCUMENTED
All code is heavily commented so that code reveiw is easy and intuitive.
OBJECT ORIENTED
Almost all code written by me is object orientated, including Javascript. Front end code follows a similar organized outline to that of a Java project, allowing for debugging to be much simpler. This design works well with front end frameworks like React.
PROGRESSIVE
All my web applications are progressive web apps, meaning they are written in a responive way that supports not only devices of varying screen sizes but also devices of varying preformance. A lot of the progressive functionality is facilitated by Firebase, however much of my frontend code follows a controller + machine structure.
